# MVP受入条件

## 1. 認証

- [ ] iPhoneのブラウザからGoogleアカウントでログインできる
- [ ] 許可リストに含まれないメールではアクセスが拒否される
- [ ] ログアウトできる

## 2. プロジェクト管理

- [ ] プロジェクトを登録できる
- [ ] プロジェクト一覧が表示される
- [ ] プロジェクト詳細が表示される

## 3. タスク作成

- [ ] 携帯から自然言語で開発指示を送信できる
- [ ] 画像/スクリーンショットを添付できる
- [ ] タスクがQUEUED状態でFirestoreに保存される

## 4. タスク取得と実行

- [ ] 開発PCのLocal Workerがタスクを自動取得する
- [ ] Antigravityワークフローが開始される
- [ ] プロジェクトの解析が行われる

## 5. 計画作成と承認

- [ ] AIが仕様・実装計画を作成する
- [ ] 携帯にプッシュ通知が届く
- [ ] 携帯で計画内容を確認できる
- [ ] 承認/却下/修正依頼ができる

## 6. 実装

- [ ] 承認後、専用ブランチが作成される（devcore/{task-id}-{slug}）
- [ ] AIモデルによるコード実装が行われる
- [ ] 実装中の進捗が携帯で確認できる

## 7. テスト

- [ ] 実装後に自動テストが実行される
- [ ] テスト結果（成功数/失敗数）が携帯で確認できる
- [ ] 未実行のテストは「未実行」と明示される

## 8. レビュー

- [ ] 実装と異なるAIモデルがレビューを行う
- [ ] レビュー結果が携帯で確認できる
- [ ] 問題があれば差し戻される

## 9. 変更確認と承認

- [ ] 携帯で変更内容の要約を確認できる
- [ ] 携帯で変更ファイル一覧を確認できる
- [ ] 承認/却下/修正依頼ができる

## 10. GitHub PR

- [ ] 承認後にGitHub PRが作成される
- [ ] PR本文に要望、実装内容、テスト結果、リスクが記載される

## 11. 通知

- [ ] 承認待ちの際にプッシュ通知が届く
- [ ] タスク完了時にプッシュ通知が届く
- [ ] エラー発生時にプッシュ通知が届く

## 12. タスク管理

- [ ] タスク一覧と履歴が表示される
- [ ] 一時停止と再開ができる
- [ ] キャンセルができる
- [ ] 優先順位の変更ができる
- [ ] レビュー再実行ができる
- [ ] テスト再実行ができる

## 13. セキュリティ

- [ ] 危険操作は承認なしに実行できない
- [ ] APIキーがブラウザに露出しない
- [ ] すべての承認操作が監査ログに記録される

## 14. オフライン対応

- [ ] PCがオフラインでも指示が消えない（Firestoreに保存）
- [ ] PCがオンラインになったら処理が再開される
- [ ] 携帯に「開発PC待機中」と表示される

## 15. コスト管理

- [ ] AI利用料金が確認できる
- [ ] タスクごとのコストが表示される
- [ ] 月間累計が表示される
- [ ] 使用モデルが表示される

## 16. 復元

- [ ] 再起動後も作業状態を復元できる
- [ ] /dev-resume で中断タスクを再開できる

## 17. ドキュメント

- [ ] 導入方法が日本語で説明されている
- [ ] トラブルシューティングが用意されている
