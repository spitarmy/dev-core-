# dev-status — ステータスダッシュボード

> **スラッシュコマンド:** `/dev-status`
>
> 現在の開発状況を一目で把握できるダッシュボードを表示します。

---

## 概要

`/dev-status` は、以下の情報を統合して表示します:

1. **現在のタスク** — 進行中のタスクと進捗
2. **担当AI** — 各フェーズで使用中のモデル
3. **テスト結果** — 最新のテスト実行結果
4. **失敗・エラー** — 発生中の問題
5. **保留中の承認** — 人間の対応が必要な項目
6. **コスト情報** — トークン使用量・費用

---

## 実行手順

### Step 1: 情報の収集

`project-memory` スキルから以下をロード:

1. `current-task.json` — 現在のタスク
2. `task-history.json` — 最近のタスク履歴
3. `model-usage.json` — モデル使用状況
4. `cost-tracking.json` — コスト情報

## 自動更新

### 状態変化時の自動通知

以下のイベント発生時に自動的にステータスを更新:

| イベント | 通知 |
|---|---|
| タスク開始 | 「TASK-xxx を開始しました」 |
| フェーズ移行 | 「実装 → テスト に移行しました」 |
| テスト完了 | 「テスト完了: xx/xx パス」 |
| エラー発生 | 「エラー: {概要}」 |
| 承認待ち | 「人間の承認が必要です」 |
| タスク完了 | 「TASK-xxx が完了しました」 |

---

## 注意事項

- ステータス表示は読み取り専用（変更操作は他のスキルで行う）
- 大量のタスク履歴がある場合は直近10件のみ表示
- コスト情報は推定値であることを注記する
- エラーの詳細は `/dev-fix` で確認・修正する
